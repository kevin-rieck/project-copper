# Domain Glossary

- **Connection**: The physical or network transport layer used to reach hardware. Examples include a Serial Port (e.g., `COM3`, `/dev/ttyUSB0`) or a TCP socket (e.g., `192.168.1.50:502`). In the UI, Connections are abstracted and implicitly managed by the backend; if multiple Devices specify the same URI, they share the same Connection.
- **Device**: The logical entity communicated with over a Connection. Uniquely identified by a `Slave ID` or `Unit ID`. Contains the default `ByteOrder` (Endianness) for decoding data. Multiple Devices can share a single Connection (common in Modbus RTU). Users interact primarily with Devices.
- **ModbusTable**: One of the four primary Modbus memory regions: Coils (0x), Discrete Inputs (1x), Input Registers (3x), or Holding Registers (4x).
- **RegisterMap**: The configured layout of a Device's ModbusTables, RegisterGroups, and RegisterDefinitions. It describes what data points the application knows how to poll and decode for that Device.
- **RegisterGroup**: A logical collection of registers (e.g. "Drive Controllers") that are typically polled and managed together. Forms the leaf nodes of the RegisterMap tree. A RegisterGroup has a stable generated identity and an editable user-facing name that must be unique within a Device. A RegisterGroup strictly belongs to exactly one ModbusTable.
- **RegisterDefinition**: The configuration for an individual data point. It defines the starting address, name, data type (e.g., Float32, UInt16), physical address count, and how to decode the raw bytes. A RegisterDefinition is edited and removed as a unit, even when it spans multiple physical addresses. It has a stable generated identity while still enforcing a unique, non-overlapping address range within a RegisterGroup. It can optionally override the Device's default `ByteOrder`; an omitted RegisterDefinition byte order means the Device default applies. Within a single RegisterGroup, RegisterDefinitions must not duplicate or overlap address ranges. Note: We use this instead of "Watch" to avoid collision with the UI's "Watch List" feature.
